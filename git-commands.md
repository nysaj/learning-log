# Git/GitHubのコマンド学習記録

## ローカルリポジトリ（自分のパソコンの保管場所）
* `git init`
    Gitで管理したいディレクトリ（フォルダ）内で最初に一度だけ実行
* `git status`
    今どのファイルが変更されたか、コミット（記録）の準備ができているかの状態を表示
* `git add [ファイル名]`
    ファイル名をステージングエリア（記録する対象リスト）に登録
* `git add .`
    変更したすべてのファイルを、ステージングエリア（記録する対象リスト）に登録（ドットは今いるフォルダ以下全部という意味）
* `git commit -m "[メッセージ]"`
    ステージングエリア（記録する対象リスト）に登録したファイルを、メッセージ付きでコミット（記録）する

## リモートリポジトリ（GitHub）との連携・送信（プッシュ）
* `git remote add origin [GitHubのURL]`
    ローカルリポジトリ（自分のパソコンの保管場所）と、GitHubのリモートリポジトリ（保管場所）を紐付ける
* `git branch -M main`
    メインの記録のブランチ（分岐）の名前を「main」にする
* `git push -u origin main`
    ローカルリポジトリの中身（セーブデータ）をGitHubに送信（初回）
* `git push`
    新しく追加したローカルリポジトリの中身（セーブデータ）をGitHubへ送信（2回目以降）
