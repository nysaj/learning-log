# Git/GitHubのコマンド学習記録

## ローカルリポジトリ（自分のパソコンの保管場所）の作成（最初に一度だけ）
* `git init`
    Gitで管理したいディレクトリ（フォルダ）内で最初に一度だけ実行
* `git branch -M main`
    メインの記録のブランチ（分岐）の名前を「main」にする

## 変更内容を確認・記録
* `git status`
    今どのファイルが変更されたか、コミット（記録）の準備ができているかの状態を表示
* `git add [ファイル名]`
    ファイル名をステージングエリア（記録する対象リスト）に登録
* `git add .`
    変更したすべてのファイルを、ステージングエリア（記録する対象リスト）に登録（ドットは今いるフォルダ以下全部という意味）
* `git commit -m "[メッセージ]"`
    ステージングエリア（記録する対象リスト）に登録したファイルを、履歴としてメッセージ付きでコミット（記録）する

## リモートリポジトリ（GitHub）との連携（最初に一度だけ）
* `git remote add origin [GitHubのURL]`
    ローカルリポジトリ（自分のパソコンの保管場所）と、GitHubのリモートリポジトリ（保管場所）を紐付ける
* `git branch`
    現在のブランチ名を確認

## GitHubへ送信
* `git push -u origin main`
    ローカルリポジトリの中身（セーブデータ）をGitHubに送信（初回）
* `git push`
    新しく追加したローカルリポジトリの中身（セーブデータ）をGitHubへ送信（2回目以降）

## コミットメッセージのルール
* `feat` 新しい機能を追加したとき
* `fix` バグ修正
* `docs` Markdown/説明書/コメント
* `style` 動作に関係ない修正（見た目）
* `refactor` 動きは変えず中身を改善
* `chore` アプリの動作に直接関係ない作業・設定変更
